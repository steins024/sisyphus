## 5. Definition File Examples

### 5.1 Tool Definition File (\`config/tools/read_file.yaml\`)

```yaml
name: read_file
description: Read the contents of a file from the filesystem
parameters:
  type: object
  properties:
    filePath:
      type: string
      description: The absolute path to the file to read
    startLine:
      type: integer
      description: The line number to start reading from (1-based)
    endLine:
      type: integer
      description: The line number to end reading at (1-based, inclusive)
  required:
    - filePath
    - startLine
    - endLine
handler: tools.filesystem.read_file
```

### 5.2 Subagent Definition File (\`config/subagents/researcher.yaml\`)

```yaml
name: researcher
description: A subagent specialized in searching and gathering information
system_prompt: |
  You are a research assistant. Your job is to search for information
  and provide comprehensive summaries. Focus on accuracy and relevance.

  - search
  - read_file
  - semantic_search
  - fetch_webpage
max_iterations: 10
```

### 5.3 Skill Definition File (\`config/skills/code-analysis/manifest.yaml\`)

```yaml
name: code-analysis
description: Analyze codebase architecture and provide insights
version: 1.0.0
author: sisyphus

# Trigger conditions - when to auto-activate this Skill
triggers:
  - keywords: ["analyze code", "code architecture", "codebase structure"]
  - intent: "code_analysis"

# Prompt file path
prompt_file: prompt.md

# Executable scripts
scripts:
  - name: analyze
    path: scripts/analyze.py
    description: Analyze code structure and dependencies
    args:
      - name: target_dir
        type: string
        required: true
  - name: report
    path: scripts/report.py
    description: Generate analysis report
    args:
      - name: format
        type: string
        default: markdown

# Skill does not register new tools, but declares required Agent tools
requires_tools:
  - read_file
  - terminal
  - search
```

\`\`\`markdown
# Code Analysis Skill

When performing code analysis, follow these steps:

1. **Discover Structure**: Use \`read_file\` to examine key files like
   package.json, requirements.txt, or build files.

   \`\`\`
   python /path/to/skills/code-analysis/scripts/analyze.py --target_dir <dir>
   \`\`\`

3. **Identify Patterns**: Look for:
   - Design patterns used
   - Dependency relationships

4. **Generate Report**: Use the report script to create a summary:
   \`\`\`
   python /path/to/skills/code-analysis/scripts/report.py --format markdown
   \`\`\`

5. **Present Findings**: Summarize the architecture in a clear,
   structured format for the user.

## 7. Key Design Principles

1. **Configuration-Driven** - Tool/Subagent/Skill all defined via config files for extensibility
2. **Unified Registration** - All components managed through a unified Registry
3. **Lazy Loading** - Skill Prompts are injected only when activated
4. **Permission Isolation** - Subagent only gets the tool subset allowed by config
5. **Lightweight Skills** - Skills don't register new tools, achieve functionality by composing existing tools
